소개

프로젝트명 : SPACE STORE

홈페이지와 제출한 자료는 개인 담당 영역 기준으로 간략하게 소개되어 있습니다.
프로젝트 전반적인 내용과 더 상세한 소개는 PDF와 GITHUB README에 준비되어 있습니다.

담당 영역소개

  • 메인페이지
    1. 메인페이지, 상세페이지, 디자인
    2. 조건에 따른 동적 조회가 가능
    3. 플랫폼에 등록된 모든 연습실 조회가능
  • 상세페이지
    1. 모든 페이지는 관리자 또는 호스트가 변경한 내용들이 실시간 반영
    2. 사용자가 원하는 방과 날짜를 자유롭게 선택하면 동적으로 예약 가능한 시간이 확인되고 원하는 시간에 예약 가능
    3. 달력 위젯을 사용해 선택할 수 있는 기간을 제한하고 무분별한 예약을 막을 수 있도록 손쉽게 구현
    4. Kakao 지도 API를 사용해서 호스트가 등록한 주소로 연습실 위치를 즉시 시각적으로 확인 가능

메인 페이지 검색하는 모습

클릭과 입력 순서 상관없이 사용자가 원하는 옵션으로 조회 가능합니다.

상세페이지 예약하는 모습

사용자가 방 번호와 날짜를 자유롭게 클릭하면
ajax를 사용해 이미 예약된 날짜와 예약 가능한 날짜 즉시 확인 가능합니다.

프로젝트 분석

  • 분석
    • 1차 프로젝트에서 아쉬웠던 프론트를 조금 더 배웠고 적용해보면서
    사용자가 웹페이지 안에서 원하는 것을 선택하면 DB에서 원하는 정보를 제공하고
    서로 소통하는 시스템을 만들 수 있다는 것이 너무 만족스러웠습니다.
  • 어려웠던 점
    • 데이터를 즉시 조회하고 사용자에게 제공해야 하는 부분들이 어려웠습니다.
    먼저는 검색 부분이었는데 페이지를 나가거나 새로 고침 하지 않더라도 자유롭게 선택 후 검색하면
    즉시 데이터를 제공하도록 구현해야 했는데 입력한 데이터를 모두 저장해두고 검색을 누르면
    선택한 옵션에 따라 SQL 문장을 동적으로 구성해서 DB를 조회하도록 했습니다.

    예약 부분에서는 이미 예약된 부분은 사용자가 예약하지 못하도록 구현해야 했는데
    예약 시 데이터 입력 순서를 고민해보고 그 순서에 따라 추가적인 정보들을 사용자에게 제공하도록 유도해서
    마지막에 예약시간을 JSTL 태그를 활용 반복문을 통해 이미 예약된 정보와 대조해서 예약하지 못하도록 표시하고 구현했습니다.
  • 개선사항
    • 사용자와 시스템이 소통할 수 있도록 돕는 것이 뿌듯했지만
    사용 기술이 많아질수록 기술과 기술을 이어주는 부분에 있어서 많이 아쉬웠던 것 같습니다.
    예를 들어 데이터를 주고받는 부분들에서 미흡함을 많이 느꼈고
    많은 기술이 쓰일수록 도움도 받고 서로 협업을 하기 위해서
    코드의 정리나 분류 등이 미흡해서 아쉬움을 많이 느꼈습니다.
